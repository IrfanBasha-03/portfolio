VPC with Public-Private Subnet

This production-ready VPC design ensures high availability and security by distributing resources across multiple Availability Zones and using both public and private subnets. It includes NAT gateways, an Application Load Balancer, and auto-scaling EC2 instances.

Technologies Used

  • AWS VPC, Subnets, Route Tables
  • EC2 with Auto Scaling
  • ALB (Application Load Balancer)
  • NAT Gateway

Challenges Faced

  • Routing public and private subnet traffic securely.
  • High-availability NAT Gateway setup across AZs.
  • Designing fault-tolerant infrastructure.

Sample Terraform Code Snippet


resource "aws_vpc" "main" {
  cidr_block = "10.0.0.0/16"
}

resource "aws_subnet" "public" {
  vpc_id = aws_vpc.main.id
  cidr_block = "10.0.1.0/24"
  map_public_ip_on_launch = true
  availability_zone = "ap-south-1a"
}
